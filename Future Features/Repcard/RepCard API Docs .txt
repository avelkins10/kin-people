This endpoint makes an HTTP GET request to retrieve minimal user information based on the provided company ID and pagination. The response will be in JSON format with a status code of 200. The "result" object also provides metadata such as totalCount, currentPage, and totalPages.

PUT
Update User
https://app.repcard.com/api/users/1466
This endpoint allows you to update user information by sending an HTTP PUT request to the specified URL.
Request Body
The request should include a raw JSON body with the following parameters:
	•	firstName (string): The first name of the user. (Required)
	•	lastName (string): The last name of the user. (Required)
	•	userEmail (string): The email address of the user. (Required)
	•	phoneNumber (number): The phone number of the user. (Required)
	•	jobTitle (string): The job title of the user. (Required)
	•	roleName (string): The role name of the user. (Required)
	•	officeName (string): The office name of the user. (Required)
	•	teamName (string): The office team name of the user. (Required)
	•	countryCode (string): The country code of the user. (Required)
	•	externalId (string): The external ID of the user (optional).
Response
The endpoint returns a status code of 401 and a JSON response with the following structure:
	•	status (number): The status of the response.
	•	statusCode (number): The status code of the response.
	•	message (string): A message related to the response.
	•	result (object): An empty object.
Note: The response indicates that the request was not authorized.

PUT
Activate/Deactivate
https://app.repcard.com/api/users/26682/activate-deactivate
Activate/Deactivate User
This endpoint allows you to activate or deactivate a user by updating their status.
Request
	•	Method: PUT
	•	Endpoint: https://app.repcard.com/api/users/{userId}/activate-deactivate
Path Parameters
	•	userId (required): The unique identifier of the user you wish to activate or deactivate.
Request Body
The request body must be in JSON format and should contain the following parameter:
	•	status (integer, required):
	◦	A value of 1 to activate the user.
	◦	A value of 0 to deactivate the user.
Example Request Body:
json
{
  "status": 1
}
Response
The response will be in JSON format and will include the following fields:
	•	status (integer): Indicates the success or failure of the request.
	•	statusCode (integer): A code representing the status of the operation.
	•	message (string): A message providing additional context about the request outcome.
	•	result (object): An object that may contain additional data related to the request.
Example Response
json
{
  "status": 0,
  "statusCode": 0,
  "message": "",
  "result": {}
}
Error Handling
A response with a 400 status code indicates a bad request, which could be due to an invalid status value or other issues with the request.
Ensure that the status parameter is correctly set to either 1 or 0 to avoid errors.
